## 一、新增基础状态

### 需要加入 BaseStateMachine 的状态

| 状态 | 动画文件 | speedK | playOnce | 时长(秒) | 环境前置条件 |
|------|---------|--------|----------|---------|-------------|
| lean_wall | `lean_wall` | 0 | true (定格末帧) | [8, 20] | `envQuery.isNearWall(npc)` = true |
| squat | `squat` | 0 | true (定格末帧) | [5, 15] | 无 |
| sit_ground | `sit_ground`* | 0 | true (定格末帧) | [8, 20] | 附近没有空长椅时优先触发 |
| lie_bench | `lie_bench` | 0 | true (定格末帧) | [15, 40] | 必须已经在 sit_bench 状态 + 持续足够久后小概率转入 |
| get_up | `get_up` | 0 | true | null (动画结束驱动) | 作为 lie_ground → stand 的过渡动画 |

*注：如果 `assets/animations/` 里没有 `sit_ground.json`，暂时复用 `squat.json` 动画，加一个 TODO 注释标记待制作。

### 状态转换变更

在 BaseStateMachine 中修改/新增以下转换逻辑（参照 npc.md 中"状态转换规则"章节）：

**新增入口转换：**

```
walk → lean_wall
  条件：前人行道行人 + envQuery.isNearWall(npc) + 随机（低概率）
  注意：公园行人不会触发（公园里没墙）

walk → squat
  条件：随机触发（低概率，约 0.05 权重）
  
stand → sit_ground
  条件：随机触发（极低概率）+ envQuery.isNearBench(npc) 返回 false（附近无长椅才坐地上）

sit_bench → lie_bench
  条件：sit_bench 持续超过 12 秒后 + 随机（极低概率，约 0.03）
  关联标签：homeless, resting, tired
```

**修改现有转换（get_up 过渡）：**

```
lie_ground → get_up → stand
  当前：lie_ground 计时结束直接跳 stand
  改为：lie_ground 计时结束 → 进入 get_up（播放起身动画）→ animDone 后进入 stand

lie_bench → sit_bench → stand
  lie_bench 计时结束 → 回到 sit_bench（短暂坐起）→ 正常走 sit_bench → stand 流程
```

**出口转换（新状态 → 回到移动）：**

```
lean_wall → stand → walk
  lean_wall 计时结束 → stand（短暂站直）→ 正常 stand → walk 流程

squat → stand → walk
  squat 计时结束 → stand

sit_ground → stand → walk
  sit_ground 计时结束 → stand
```

### EnvironmentQuery 补充

确认 `EnvironmentQuery` 提供以下方法（如果重构时已实现则跳过）：

```js
// 判断 NPC 是否靠近建筑墙面
// 条件：npc.y 在 BUILDING_BASE_Y 附近（前人行道带，Y ≈ 210~268）
//       且 npc 头顶方向有建筑实体
isNearWall(npc, threshold = 30)

// 判断附近是否有空闲长椅（未被其他 NPC sit_bench / lie_bench 占用）
isNearBench(npc, radius = 60)

// 找到附近最近的空闲长椅（用于 sit_bench 时对齐位置）
nearestFreeBench(npc, radius = 80)
```

`isNearWall` 的判定逻辑：检查 `npc.y` 是否在 `SIDEWALK_FAR_Y ± threshold` 范围内（前人行道带），且附近有 `BuildingEntity`。公园区域（Y > NEAR_Y）的 NPC 永远返回 false。

### phone_call

```
触发条件：随机（概率略低于 phone_look，约 0.002/帧）
可叠加基础状态：stand, walk, sit_bench, lean_wall
持续时长：[10, 20] 秒
标签：phone_call
```

### smoke

```
触发条件：仅限 profile.traits 中有 smoker: true 的 NPC
          随机（概率约 0.002/帧）
          lean_wall 时概率翻倍（靠墙抽烟最自然）
可叠加基础状态：stand, lean_wall, sit_bench
持续时长：[15, 30] 秒
标签：smoke, smoking
视觉：TODO（后续实现 overlayPose：一只手举到嘴边）
当前只设标志位 npc.overlay = 'smoke'
```

### hold_bag（持久特征，非 overlay）

```
实现方式：不走 OverlayLayer 的随机触发/消失逻辑
         在 NPC 生成时（Pedestrians.js spawner 里）按概率给 NPC 设置
         npc.persistentOverlay = 'hold_bag'
         
         OverlayLayer 检查：如果 npc.persistentOverlay 存在，
         在没有其他 overlay 激活时，npc.overlay 回退到 persistentOverlay
         （即 phone_call 结束后恢复为 hold_bag，而不是 null）

触发概率：约 30% 的 pedestrian / 50% 的 businessman 在生成时携带
可叠加基础状态：walk, run, stand（其他状态下暂时隐藏）
标签：hold_bag, carrying
视觉：TODO（后续实现 drawExtra：手部垂一个矩形包）
当前只设标志位
```

### OverlayLayer 修改

```js
function tickOverlay(npc, profile, dt) {
  // 1. 如果当前有活跃 overlay（非持久），tick 计时
  //    计时结束 → npc.overlay = npc.persistentOverlay || null
  
  // 2. 如果当前无活跃 overlay（或只有持久的）
  //    遍历 profile.overlays 检查：
  //      - 当前基础状态是否在 overlay 的兼容列表里
  //      - trait 前置条件是否满足（smoke 需要 smoker trait）
  //      - 按概率随机触发
  //    触发时设 npc.overlay = overlayName, npc._overlayTimer = dur
  
  // 3. 持久 overlay 回退逻辑：
  //    如果 npc.persistentOverlay 存在
  //    且 npc.overlay 为 null
  //    且当前基础状态在 hold_bag 的兼容列表里
  //    → npc.overlay = npc.persistentOverlay
}
```

## 三、Profile 调整

### pedestrian

```js
{
  name: 'pedestrian',
  allowedStates: ['walk', 'run', 'stand', 'sit_bench', 'fall', 'lie_ground',
                  'squat', 'sit_ground', 'get_up'],
  // 注：不含 lean_wall（公园行人远离建筑）
  // 前人行道行人 spawner 里单独覆盖 allowedStates 加入 lean_wall
  transitions: {
    walk:       { stand: 0.55, sit_bench: 0.2, run: 0.08, squat: 0.05, sit_ground: 0.02 },
    // 注：sit_bench 需要 envQuery.isNearBench 前置检查，不满足则 fallback 到 stand
    // 注：sit_ground 需要 !envQuery.isNearBench 前置检查（附近没椅子才坐地上）
    run:        { walk: 0.9, fall: 0.1 },
    stand:      { walk: 0.85, sit_bench: 0.1, sit_ground: 0.03, squat: 0.02 },
    sit_bench:  { stand: 0.97, lie_bench: 0.03 },
    // 注：lie_bench 需要 sit_bench 已持续超过 12 秒
    squat:      { stand: 1.0 },
    sit_ground: { stand: 1.0 },
    fall:       { lie_ground: 1.0 },
    lie_ground: { get_up: 1.0 },
    get_up:     { stand: 1.0 },  // animDone 驱动，不用计时
    lie_bench:  { sit_bench: 1.0 },
  },
  overlays: {
    phone_look: { on: ['walk', 'stand'], chance: 0.004, dur: [5, 25] },
    phone_call: { on: ['walk', 'stand', 'sit_bench'], chance: 0.002, dur: [10, 20] },
  },
  // smoke 不在这里，因为需要 smoker trait；有 trait 的 NPC 在 spawner 里覆盖 overlays
  activities: ['talk'],
  traits: {},
}
```

### pedestrian_sidewalk（前人行道行人的 profile 覆盖）

两种实现方式任选：
- **方案 A**：新建一个 `pedestrian_sidewalk` profile，继承 pedestrian 然后加入 lean_wall
- **方案 B**：pedestrian profile 里加入 lean_wall，但 BaseStateMachine 用 `envQuery.isNearWall()` 自然过滤（公园 NPC 永远不满足条件所以不会触发）

推荐**方案 B**，更简洁。即：在 pedestrian 的 allowedStates 和 transitions 里加入 lean_wall，由环境条件自然控制。

```js
// 方案 B 的话，pedestrian.allowedStates 加入 'lean_wall'
// transitions 加入：
stand: { walk: 0.80, sit_bench: 0.1, sit_ground: 0.03, squat: 0.02, lean_wall: 0.05 },
// lean_wall 需要 envQuery.isNearWall 前置检查，不满足则 fallback 到 stand
lean_wall: { stand: 1.0 },
```

### smoker trait

不是独立 profile，而是在 spawner 里给部分 NPC 注入 trait + overlay：

```js
// Pedestrians.js 里，约 15% 的 NPC 生成时：
npc._profile = { ...getProfile('pedestrian') };  // 浅拷贝
npc._profile.traits = { smoker: true };
npc._profile.overlays = {
  ...npc._profile.overlays,
  smoke: { on: ['stand', 'lean_wall', 'sit_bench'], chance: 0.002, dur: [15, 30],
           traitRequired: 'smoker', chanceMultiplier: { lean_wall: 2.0 } },
};
```

或者更优雅的做法：Profile 里统一定义 smoke overlay 并标注 `traitRequired: 'smoker'`，OverlayLayer 在触发前检查 trait。推荐后者。

## 七、验证标准

1. **公园行人**（Y > 333）：能触发 walk / run / stand / sit_bench / squat / sit_ground / fall → lie_ground → get_up → stand 全流程；sit_bench 长时间后偶尔进入 lie_bench
2. **前人行道行人**（Y ≈ 240）：能触发 lean_wall（靠墙站），公园行人**不能**触发 lean_wall
3. **smoker trait NPC**：在 stand / lean_wall / sit_bench 时偶尔触发 smoke overlay；lean_wall 时概率更高；非 smoker NPC 绝不触发
4. **phone_call overlay**：在 stand / walk / sit_bench 时偶尔触发，和 phone_look 互斥（不会同时出现两个 overlay）
5. **hold_bag 持久 overlay**：部分 NPC 生成后始终显示 hold_bag（walk/stand 时），phone_call 激活时临时替代，结束后恢复 hold_bag
6. **get_up 过渡**：lie_ground 结束后必须先播放 get_up 动画再进入 stand，不再跳跃式切换
7. **标签输出**：取景框捕获 NPC 时 debug overlay / console 显示的标签集合应包含新状态标签
8. **现有行为不退化**：talk 配对、漫游转向、横穿者、棋手、遛狗、运动者全部正常
