# 视觉设计规范

> 适用范围：所有 `draw*.js` 和 `SceneRenderer.js`
> 风格定位：黑白灰线稿 + 轻体积感，火柴人 NPC 不变

---

## 铁律（不可违反）

1. **禁止魔法颜色** — 填充只用 `FILL_PAPER / FILL_LIGHT / FILL_MID / FILL_SHADE`，线条只用 `depthLineColor(y, {light: ENV_LINE_LIGHT, dark: ENV_LINE_DARK})`。硬编码 `0x??????` 即违规。
2. **`lineStyle(0)` 入口重置** — 每个 draw 函数第一行。
3. **`depthT` 是唯一深度源** — 线粗、线色、缩放全部从它派生，禁止第二套公式。

---

## 色彩

四档填充常量，不增不减：

| 常量 | 角色 |
|------|------|
| `FILL_PAPER` | 最亮面 |
| `FILL_LIGHT` | 次亮面 |
| `FILL_MID` | 中灰面 |
| `FILL_SHADE` | 最暗面 |

需要比四档更细的差异时用 alpha 调节，不发明新常量。唯一允许的非常量色是 `0x000000` 和 `0xffffff`（低 alpha 叠加用）。

---

## 线条

所有环境线通过统一的 `lenv(g, baseY, wScale)` 函数设置。`wScale` 分三档用途：

| wScale 范围 | 用途 |
|-------------|------|
| 0.85 – 1.0 | 外轮廓 |
| 0.5 – 0.65 | 内结构（窗框、面板分隔） |
| 0.3 – 0.4 | 细节纹理 |

`depthLineColor` 的 light/dark 参数必须使用 `ENV_LINE_LIGHT` / `ENV_LINE_DARK`，不许传自定义值。

---

## 体积

光源假设**左上方**。所有有体积的实体用三面法表达立体感：

| 面 | 取值 |
|----|------|
| 顶面 | 比正面亮一档 |
| 正面 | 基准色 |
| 侧面（右侧） | 比正面暗一档 |

两种材质色阶供选择：

- **浅色材质**：顶 PAPER / 正 LIGHT / 侧 MID
- **深色材质**：顶 LIGHT / 正 MID / 侧 SHADE

约束：
- 侧面深度 `D ≈ W * 0.2`（0.15–0.25 范围内）
- 顶面纵向缩短 `= D * 0.6`（统一俯视角）
- 太窄/太小的部件（杆、柱、线条类）免除三面

不适合三面的形体（树冠等球形/有机形态）用明暗半球法：整体 `FILL_MID` + 偏左上叠一层 `FILL_LIGHT`。

---

## 阴影

每个站在地面上的实体画地面投影椭圆：

- 填充：`0x000000, alpha 0.12`（全场统一）
- 尺寸：`radiusX = 实体宽度 * 0.55`，`radiusY = radiusX * 0.3`
- 地面道具（井盖、排水口等平贴地面的）不加阴影

---

## 绘制顺序

```
地面阴影 → 侧面 → 正面 → 顶面 → 内部细节 → 轮廓线（最后）
```

轮廓线永远最后画，压在所有填充之上。

---

## 不在本规范范围内

- NPC / 火柴人渲染（StickRenderer 不动）
- 地面分区色值（GRAY_* 不调）
- 动画系统 / assets 重构
- 新闻生成管线
