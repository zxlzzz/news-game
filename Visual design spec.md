# 视觉设计规范

> 适用范围：所有 `draw*.js` + `SceneRenderer.js`
> 视角：**侧视图**（2000×520px 横向卷轴），不是等轴投影。三面法是在侧视基础上加伪 3D 装饰——给人"稍微能看到顶面和右侧面"的暗示，不是真正的 2.5D
> 风格定位：黑白灰线稿 + 轻体积感。线条是主角，体积是辅助
> `s`：实体缩放系数（`p.scale ?? 1`），通常基于深度位置。所有像素尺寸乘以 s 以适配不同深度下的缩放
> 本文件定义原则和判断标准，不涉及具体实施代码

---

## 一、铁律

1. **禁止魔法颜色** — 填充只用 `FILL_PAPER / FILL_LIGHT / FILL_MID / FILL_SHADE`，线条只用 `depthLineColor(y, {light: ENV_LINE_LIGHT, dark: ENV_LINE_DARK})`。出现硬编码 `0x??????` 即违规。**显式例外**：`0x000000` 低 alpha（阴影叠加）和 `0xffffff` 低 alpha（高光线/反射），这两者允许手写 `lineStyle` / `beginFill`，不走 `lenv`。
2. **`lineStyle(0)` 入口重置** — 每个 draw 函数第一行。
3. **`depthT` 是唯一深度源** — 线粗、线色、缩放全部从它派生，禁止第二套公式。

---

## 二、色彩系统

### 2.1 四档常量

| 常量 | 角色 |
|------|------|
| `FILL_PAPER` | 最亮面 |
| `FILL_LIGHT` | 次亮面 |
| `FILL_MID` | 中灰面 |
| `FILL_SHADE` | 最暗面——深色材质侧面，也用于凹陷/内部空间（门洞深处、电话亭内部、出货口、窗洞等"凹进去"的暗面） |

需要更细的差异时用 alpha 调节，不发明新常量。

### 2.2 材质区分

四档灰度不可能同时区分所有材质。**通过纹理密度区分材质，而非灰度**：

| 材质 | 纹理处理 |
|------|----------|
| 砖/石 | 填充面上叠加细横线（最细档线宽，极低 alpha） |
| 木材 | 断续浅弧线/纹理 |
| 金属 | 转折处加一条极细 `0xffffff` 高光线 |
| 玻璃 | `0xffffff` 低 alpha 矩形叠加作反射 |
| 植物 | 明暗半球法（见体积系统） |

灰度体系完全不变，材质辨识度靠纹理拉开。

## 三、线条系统

所有环境线通过统一的 `lenv(g, baseY, wScale)` 函数设置。

### 3.1 wScale 层级

| wScale | 用途 |
|--------|------|
| 0.85 – 1.0 | 外轮廓 |
| 0.5 – 0.65 | 内结构（窗框、面板分隔） |
| 0.3 – 0.4 | 细节纹理、地面纹理 |

### 3.2 wScale 与 depthT 的关系

最终线宽 = `depthLineWidth(baseY) * wScale`。`depthLineWidth` 随 Y 坐标变化（远处细、近处粗），`wScale` 在此基础上做**同一实体内部**不同层级的相对缩放。两者是乘法关系，不会冲突。远景线条不会消失，因为 `depthLineWidth` 有下限 `wMin: 0.5`。

### 3.3 禁止事项

- `depthLineColor` 必须使用 `ENV_LINE_LIGHT / ENV_LINE_DARK`，不许传自定义 light/dark 值
- 不许手动写 `g.lineStyle(1.5, 0x606060, 1)` 跳过 lenv
- **例外**：`0xffffff` 高光线（金属转折、玻璃边缘）允许手写 `g.lineStyle(w, 0xffffff, alpha)` 不经过 lenv，因为高光线的颜色和深度无关。这与铁律 1 的显式例外一致

## 四、体积系统

光源假设**左上方**。

### 4.1 三面法

有体积的实体用三面法表达立体感：

| 面 | 取值 |
|----|------|
| 顶面 | 比正面亮一档 |
| 正面 | 基准色 |
| 侧面（右侧） | 比正面暗一档 |

两种材质色阶：

- **浅色材质**：顶 PAPER / 正 LIGHT / 侧 MID
- **深色材质**：顶 LIGHT / 正 MID / 侧 SHADE

每个独立体块各自有侧面和顶面。

### 4.2 侧面参数

- **侧面深度 `D`**：D = W * 0.2（范围 0.15–0.25），W 为该体块正面在屏幕上的像素宽度（本游戏是侧视图，不是等轴投影，不需要做等距校正）
- **顶面纵向缩短**：顶面高度 = D * 0.6（统一俯视角约 30°）

### 4.3 体积阈值（关键）

**宽度 < 16\*s 的部件降级为线条 + 单色填充，不画三面。**

这意味着：杆、柱、支架、出水口、椅腿、灯杆、挂杆等细长部件全部保持线条风格。只有主体足够宽的体块才画三面。

判断标准：**如果加了侧面后侧面只有几个像素宽，画出来只会像渲染错误而不是体积感，那就不该画。**

### 4.4 不适合三面的形体

- 球形/有机形体（树冠、灌木）→ 明暗半球法：整体 `FILL_MID` + 偏左上叠 `FILL_LIGHT`
- 薄片/挂件（标牌 sign）→ 不画侧面和顶面，保持平面
- 太细的部件（见 4.3）→ 降级为线条

### 4.5 什么时候不该画

三面法是手段不是目的。**如果一个场景里所有体块都严格三面，大量灰面会挤压线条的存在感，画面从线稿滑向低多边形灰模。** 要维护线稿的轻灵感，应当克制使用体积：

- 大件（建筑、贩卖机、电话亭）：完整三面
- 中件（邮箱、垃圾桶、报架）：简化三面
- 小件、细件、薄片：线条 + 单色
- 心存疑虑时，少画一面好过多画一面

## 五、阴影系统

### 5.1 软阴影

每个站在地面上的实体脚下画**多层递减 alpha 椭圆**模拟软边阴影（不用 BlurFilter，不改架构）：

| 层 | alpha | 半径倍数 |
|----|-------|----------|
| 内层 | 0.08 | 1.0× |
| 中层 | 0.05 | 1.3× |
| 外层 | 0.03 | 1.6× |

### 5.2 方向偏移

光源左上方 → 阴影向**右下偏移**。偏移量：

```
offsetX = +radiusX * 0.15
offsetY = +radiusY * 0.25
```

阴影中心 = (实体x + offsetX, 实体y + offsetY)。不是正下方。

### 5.3 形状

阴影形状应当**粗略匹配物体轮廓**，不是所有物体都用同一个椭圆比例：

- 方形物体（贩卖机、电话亭）→ 扁椭圆，radiusY ≈ radiusX * 0.3
- 圆形物体（喷泉）→ 接近圆形的椭圆，radiusY ≈ radiusX * 0.5
- 长条形物体（长椅）→ 极扁椭圆，radiusY ≈ radiusX * 0.15

### 5.4 不加阴影的情况

- 地面道具（井盖、排水口）
- 悬空/挂墙道具（标牌 sign）
- 已经有其他遮蔽的（公交站下的长椅不需要独立阴影，被顶棚遮了）

## 六、绘制顺序

### 6.1 实体间排序

按实体**接地点的屏幕 Y 坐标**从远到近排序绘制。远处的先画，近处的后画覆盖。悬空物体（标牌、路灯顶部）使用它所附属的地面位置 Y 参与排序。

### 6.2 实体内部分层

在每个**体块**内部，按以下顺序：

```
侧面填充 → 正面填充 → 顶面填充 → 细节/纹理 → 该体块的轮廓线
```

地面阴影在整个实体的所有体块之前画（因为阴影在最底层）。

**轮廓线在每个体块完成填充后立即绘制**，不是实体全局最后统一画。这样多体块叠加时（底座+主体+顶部），每个体块的轮廓线只压在自己的填充上，不会覆盖其他体块。

完整流程：

```
地面阴影 → 体块A（侧→正→顶→细节→轮廓） → 体块B（侧→正→顶→细节→轮廓） → ...
```

**重点：这是每个实体各自走一遍的完整流程，不是全局分层。** 不能把所有物体的阴影画完再画所有侧面——那样前后遮挡关系会穿帮。

## 七、地面纹理

| 区域 | 纹理 | 参数 |
|------|------|------|
| 人行道 | 细横线模拟铺装接缝 | 间距 ≈ 20*s，最细档线宽，alpha 0.06 |
| 路面 | 无纹理 | 保持纯色 |
| 公园 | 零散短斜线模拟草 | alpha 0.05，最细档线宽 |

成本极低，但空间感会扎实很多。

## 八、NPC 融合

火柴人不改 StickRenderer，但需要两处微调让 NPC 融入同一空间逻辑：

1. **脚底阴影点** — 每个 NPC 脚下加一个极小的阴影椭圆（同样多层软边，radiusX ≈ 5*s）
2. **线宽跟随 depthT** — NPC 的 stroke 线宽做轻微的深度缩放（远处稍细、近处稍粗）

---

## 九、视觉聚焦

等距/侧视场景天然视觉均匀。允许每个场景 1–3 个聚焦物体：

- 轮廓线 wScale 提高 0.1–0.15
- 或填充对比度提高一档（本来用浅色材质的改用深色材质）

不引入任何颜色，单靠粗细和灰度微调建立视觉引导。具体哪些物体聚焦由场景设计决定，不在 draw 文件层面写死。
