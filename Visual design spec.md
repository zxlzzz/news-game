# 视觉设计规范

> 适用范围：所有 `draw*.js` + `SceneRenderer.js`
> 视角：侧视图（2000×520px 横向卷轴）
> 风格：纯 2D 平面黑白灰线稿。不做伪 3D，不加阴影
> `s = p.scale ?? 1`，基于深度位置的实体缩放系数

---

## 铁律

1. **禁止魔法颜色** — 填充只用 `FILL_PAPER / FILL_LIGHT / FILL_MID / FILL_SHADE`。线条只用 `depthLineColor(y, {light: ENV_LINE_LIGHT, dark: ENV_LINE_DARK})`。**例外**：`0x000000` 和 `0xffffff` 低 alpha（纹理叠加/高光/反射），允许手写 lineStyle/beginFill。
2. **`lineStyle(0)` 入口重置** — 每个 draw 函数第一行。
3. **`depthT` 唯一深度源** — 线粗、线色、缩放全部从它派生。

---

## 色彩

四档常量，不增不减：

| 常量 | 用途 |
|------|------|
| `FILL_PAPER` | 最亮——主要面、高光区域 |
| `FILL_LIGHT` | 次亮——次要面、玻璃、窗户 |
| `FILL_MID` | 中灰——屋顶、暗色部件、植物暗面 |
| `FILL_SHADE` | 最暗——凹陷内部（门洞、出货口、窗洞深处）、最暗部件 |

需要更细差异时用 alpha 调节。

**例外——建筑调色板 `BUILDING_PALETTE`**：背景板需要多档灰度区分不同楼体风格，允许在四档之外定义额外灰值，但必须集中定义于 `js/entity/building/building.js` 顶部的 `BP_*` 常量组，禁止散落在其他文件或样式表字面量中。

---

## 线条

统一通过 `lenv(g, baseY, wScale)` 设置。最终线宽 = `depthLineWidth(baseY) * wScale`，下限 `wMin: 0.5` 保证远景不消失。

| wScale | 用途 |
|--------|------|
| 0.85 – 1.0 | 外轮廓 |
| 0.5 – 0.65 | 内结构 |
| 0.3 – 0.4 | 细节纹理 |

`depthLineColor` 的 light/dark 必须用 `ENV_LINE_LIGHT / ENV_LINE_DARK`。高光线（`0xffffff`）是例外，不走 lenv。

---

## 材质区分

四档灰度无法区分所有材质。**靠纹理密度区分，不靠灰度**：

| 材质 | 方式 |
|------|------|
| 砖/石 | 填充面上叠加细横线（最细档，alpha ≈ 0.15） |
| 木材 | 断续浅弧线 |
| 金属 | 转折处极细 `0xffffff` 高光线 |
| 玻璃 | `0xffffff` 低 alpha 矩形叠加 |
| 植物 | 明暗叠色法：底层 `FILL_MID` + 上层偏移 `FILL_LIGHT` |

---

## 锚点约定

所有 draw 函数中 `p.y` 统一为**物体最低点**（地面接触点）。物体向上生长（`y - h`）。

---

## 绘制顺序

实体间按**接地点 Y 坐标**从远到近排序。远处先画，近处覆盖。悬空物体用附属地面位置 Y 参与排序。

---

## 地面纹理

| 区域 | 纹理 |
|------|------|
| 人行道 | 极细横线模拟铺装接缝（间距 ≈ 20\*s，alpha 0.06） |
| 路面 | 无 |
| 公园 | 零散短斜线模拟草（alpha 0.05） |

---

## NPC 融合

StickRenderer 不改，但两处微调：

1. 脚底加极小阴影点（多层软边椭圆，radiusX ≈ 5\*s）
2. 线宽跟随 depthT 轻微缩放

---

## 视觉聚焦

允许每场景 1–3 个聚焦物体：轮廓 wScale +0.1–0.15 或填充对比度提高一档。不引入颜色，靠粗细和灰度微调。

---

## 不做的事

- 不画侧面、顶面、三面体积
- 不画地面投影阴影
- 不改动画系统 / assets
- 不改新闻管线
